# Agent v2 场景测试指南

## 本次改进摘要

| 改进点 | 旧行为 | 新行为 |
|-------|--------|--------|
| 首条消息上下文 | 永远 Level 0（无业务背景） | 分析消息内容，自动给 Level 1-3 |
| 实体识别 | 必须手动传 entity_ref | 从消息文字自动识别客户/商机 |
| 角色定制 | 所有人同一个 prompt | 每个团队成员有专属风格和焦点 |
| 行业知识 | 极简描述 | 详细产品线、参数、竞品、客群 |

## 通用验证点

每个场景都检查：

1. **上下文标识**：AI 回复前有没有显示 `Level X` 标签（在对话气泡上方）
2. **实体识别**：提到客户名时，AI 是否引用了该客户的具体信息（联系人、历史记录等）
3. **角色感知**：Larry 的回复应该销售感强、Gu 应该技术感强、Jamie 应该战略感强
4. **行动建议**：大多数回复末尾是否有 2-3 条具体的下一步行动
